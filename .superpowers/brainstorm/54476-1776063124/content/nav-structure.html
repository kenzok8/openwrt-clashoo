页面结构方案

当前 clashoo 有 6 个独立页面，选一个重组方向

A

精简为 2 页

首页（Dashboard）：状态卡片 + 启停 + 模式切换 + 实时日志
设置页：Tab 切换（代理 / DNS / 订阅 / 系统），所有配置收进来

🟢 结构最简洁，移动端最友好
🔴 设置 Tab 内容较多

B

保持 3 页

概览：状态 + 控制
配置：订阅管理 + 代理 + DNS（Tab 切换）
系统：内核 / GeoIP / 日志 / 自动化

🟢 职责分明，不过于拥挤
🟡 折中方案，主流选择

C

维持 5 页（精简内容）

保留原有导航结构，但每个页面只保留核心选项，把高级/边缘设置折叠隐藏

🟢 改动最小，用户习惯不变
🔴 没有从根本解决"太杂乱"问题
