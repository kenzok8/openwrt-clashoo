# 故障排查

排查时先确认三个事实：

1. 当前使用哪个内核
2. 当前使用哪个配置文件
3. 当前运行模式是 Fake-IP、TUN 还是 Mixed

## 健康检查失败

可能原因：

- DNS 没有响应
- 显式代理端口不可用
- 透明代理规则没有生效
- 节点不可用
- 配置文件不兼容当前内核

处理顺序：

1. 看 `系统 -> 日志 -> 核心日志`
2. 切回 Fake-IP 测试
3. 换一个已知可用节点或配置
4. 重启服务

## 国内能通，国外不通

常见原因：

- 节点不可用
- 策略组选到了不可用节点
- DNS 能解析，但代理出站失败
- sing-box JSON 转换后节点协议不兼容

建议：

- 切换节点
- 查看核心日志中的 `ERROR`
- 先测试显式代理，再测试透明代理

## 国外能通，国内不通

常见原因：

- 国内域名被错误代理
- 大陆白名单或 GeoIP 异常
- DNS 上游不可用
- 直连规则异常

建议：

- 更新大陆白名单和 GeoIP
- 检查 DNS 设置
- 暂时切回默认 DNS 策略

## TUN 或 Mixed 不通

先确认设备是否支持 TUN：

```text
/dev/net/tun
```

容器或 PVE LXC 环境还需要宿主机允许 TUN。

如果不确定环境是否支持，先使用 Fake-IP。

## sing-box 启动失败

常见原因：

- JSON 格式错误
- 配置来自 YAML 转换，但字段不兼容
- 入站端口冲突
- TUN 与 routing_mark 等字段冲突

建议：

- 使用 Clashoo 的配置文件编辑查看 JSON 是否格式化
- 切回 Fake-IP 测试
- 查看核心日志中的 `FATAL` 或 `ERROR`

## 核心下载失败

常见原因：

- GitHub 或镜像源不可达
- 架构选择错误
- Release 资产名称变化

建议：

- 更换镜像源
- 确认设备架构
- 查看更新日志

## 面板打不开

可能原因：

- Clashoo 服务未运行
- Dashboard 端口未监听
- 面板文件未下载
- 浏览器缓存旧资源

建议：

- 重新更新面板
- 刷新浏览器缓存
- 检查 `dash_port`

## 停止服务失败

可能原因：

- 异步任务仍在运行
- 内核进程卡住
- 更新任务占用锁

建议：

- 等待几秒后刷新状态
- 查看插件日志
- 避免连续快速点击启动 / 停止

## 配置改坏了，想回到之前的状态

如果问题是改配置或换内核之后才出现的，可以用 `系统 -> 备份与还原`：

- 之前导出过备份 —— 点「导入还原」回到那一份
- 没有备份、又找不到原因 —— 点「还原默认配置」把设置重置为出厂默认

详见 [[系统维护]]。
