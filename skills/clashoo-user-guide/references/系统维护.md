# 系统维护

`系统` 页面用于组件更新、备份还原、内核下载、面板更新、资源更新、定时任务和日志查看。

## 组件更新

`组件更新` 把内核、内核数据、插件本体拆成单独的条目，逐个检查和更新，失败时能精确定位是哪个组件出了问题。

- 点「检查更新」获取各组件的最新版本
- 每个组件单独点「更新」，互不影响
- 处理器架构会自动识别，也可手动指定

内核类组件（mihomo / sing-box / Smart）默认收在「高级设置」里，避免页面臃肿。

## 备份与还原

`备份与还原` 是折腾配置前的兜底，在「组件更新」卡片下方，有三个按钮：

- **导出备份**：把当前 UCI 设置和订阅 / 上传 / 自定义 / sing-box 配置文件，打包成一个 JSON 文件下载到本地。
- **导入还原**：选择之前导出的备份文件，一键把配置还原回那个时间点的状态。
- **还原默认配置**：把 Clashoo 的设置重置为出厂默认值（订阅和配置文件保留）。

建议在更新内核、批量改配置之前先导出一份备份。配置改坏了，导入备份即可回退；设置乱了又找不到原因，用「还原默认配置」一键回到初始状态。

> 备份只包含配置与设置，不含内核和 GeoIP 等大文件 —— 这些可以在「组件更新」里重新下载。

## 内核下载

内核下载用于更新 mihomo、Smart 或 sing-box 二进制。

常见设置：

- 内核通道：稳定版、预发布版、Smart、sing-box
- 架构：根据设备 CPU 选择
- 镜像源：GitHub 直连或镜像加速

如果下载失败，先检查：

- 路由器是否能访问 GitHub
- 镜像源是否可用
- 设备架构是否选错

## 管理面板

Clashoo 支持多个管理面板，例如 Zashboard、MetaCubeXD、YACD、Razord。

面板更新只影响 Web 管理界面，不等于更新代理内核。

## GeoIP 和规则资源

资源更新通常包括：

- GeoIP
- GeoSite
- 大陆白名单

这些资源用于 DNS、分流和直连判断。

## 定时更新

定时更新用于自动更新 Clashoo 资源。

推荐关注：

- 大陆白名单更新
- GeoIP 更新

订阅自动更新如果不需要，可以关闭，避免转换类订阅被自动覆盖。

## 清理日志

可以开启自动清理日志，避免设备存储被日志占满。

日志分区说明见 [[日志说明]]。
