# 配置文件

配置文件决定代理节点、规则、策略组和 DNS 行为。Clashoo 支持 mihomo YAML 和 sing-box JSON 两套配置。

## mihomo 配置

mihomo 通常使用 YAML 格式：

```text
xxx.yaml
```

适合：

- 机场提供 Clash / mihomo 订阅
- 用户已有 Clash Meta 配置
- 想使用 Smart 内核

常见操作：

- `更新`：重新拉取订阅
- `切换`：设为当前运行配置
- `删除`：移除配置文件

自定义上传的文件通常可以编辑。

## sing-box 配置

sing-box 使用 JSON 格式：

```text
xxx.json
```

适合：

- 机场直接提供 sing-box JSON
- 用户用外部工具转换好了 JSON
- 在 Clashoo 中把 YAML 转换为 sing-box JSON

Clashoo 会在启动前对 sing-box JSON 做运行时规则化，例如补齐端口、DNS 入站、面板 API 和透明代理相关字段。

## YAML 转 sing-box JSON

如果机场只提供 YAML，但你想使用 sing-box，可以使用转换功能。

支持转换的节点协议：

- `ss`（Shadowsocks）、`vmess`、`vless`、`trojan`、`hysteria2`（含 `hy2`）、`tuic`
- 其它协议会被跳过，不会出现在转换后的 JSON 里
- 支持订阅里的 `proxy-providers`（http 类）：转换时会先下载再合并节点

转换时 Clashoo 还会自动做两件事：

- 套用内置模板生成「节点选择 / 各地区自动 / 流媒体分流」等策略组，并按节点名识别 **港 / 日 / 美 / 新及其它** 自动分组（某地区没有节点时回退到全部节点）
- 剔除机场的**伪节点**（流量、到期、官网、客服、QQ 群等信息条目），避免它们在自动选择里以 0ms「胜出」吞掉流量

需要注意：

- 转换结果依赖原始 YAML 的节点类型和规则结构
- 并不是所有 Clash 特性都能 100% 映射到 sing-box
- 转换后建议先在 Fake-IP 模式下启动测试，再测试 TUN / Mixed

## 更新、编辑、切换、删除的区别

- `更新`：重新从订阅链接拉取内容
- `编辑`：修改本地文件内容
- `切换`：设为当前使用的配置
- `删除`：删除该配置文件

订阅文件通常不建议手动编辑，因为下次更新会覆盖修改。

## 推荐习惯

- mihomo 用户优先保留原始 YAML 订阅
- sing-box 用户保留转换后的 JSON，并记录来源
- 测试新配置前，先确认旧配置可用
- 配置异常时，先切回已知可用配置
