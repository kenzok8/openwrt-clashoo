# 仅内核

仅内核（进阶）让 Clashoo 只负责拉起内核、运行你导入的配置，**不接管防火墙 / DNS / 路由**，由配置自己完成透明代理。适合把其它插件已经调好的现成配置直接拿过来跑。

## 位置

配置 → 透明代理 → 最下方「仅内核（进阶）」勾选框。

## 作用

- 开启后：只运行你导入的配置，Clashoo 不改写代理逻辑、不下防火墙规则、不劫持 dnsmasq、不建路由表。
- 关闭后（默认）：Clashoo 正常接管，按 UCI 设置改写配置（mixin）、配置防火墙与 DNS。

效果等同 nikki 的「仅核心」。

## 双内核支持

| 内核 | 行为 |
|------|------|
| mihomo / Smart | 配置**原样运行**（逐字节不改写），适配 nikki、OpenClash 导出的 YAML |
| sing-box | 老版本格式会**自动迁移**到当前内核可用的格式，兼容 momo、homeproxy 的配置 |

> Smart 内核本质是带 Smart 功能的 mihomo，仅内核下同样吃 mihomo 配置；配置里写了 `type: smart` 策略组也能识别。

## 前提

导入的配置**必须自带透明代理**，否则 LAN 流量不会走代理：

- mihomo：`tun.auto-route: true`（自带 TUN 自动路由），或自带 redir / tproxy 入站；
- sing-box：inbound 里有 `tun`（`auto_route: true`）或 redirect / tproxy。

OpenClash、nikki、momo、homeproxy 导出的完整配置通常都是自包含的，所以能直接平移。

## 适用场景

- 从 OpenClash / nikki 迁移现成 mihomo YAML；
- 从 momo / homeproxy 迁移现成 sing-box JSON；
- 直接运行一份在别处调试好的完整配置，不想被 Clashoo 改写。

## 注意事项

- 仅内核是**全局开关**，与「当前激活哪个配置」相互独立。
- 用**自包含配置**（自带 TUN/TProxy）时开启仅内核；切回**普通机场订阅**（不自带透明代理）时记得**关闭**仅内核，让 Clashoo 接管，否则流量不走代理。
- 切换仅内核后需**重启服务**才生效。
- sing-box 在仅内核下面板能否打开，取决于你的配置是否自带 `experimental.clash_api`。

## 故障排查

- 开启仅内核后不能上网：检查配置是否自带透明代理（`tun.auto-route` / tproxy 入站）；普通订阅请关闭仅内核。
- 核心起不来：多为配置与内核版本不兼容（sing-box 尤其）。Clashoo 会自动迁移老格式，仍失败时检查核心日志。
- 想恢复正常接管：关闭仅内核 → 重启服务。
