# 代理模式

Clashoo 的代理模式决定路由器如何接管流量。

## Fake-IP

推荐默认模式。

Fake-IP 会给需要代理的域名分配一个虚拟 IP，例如 `198.18.x.x`。核心收到访问这个虚拟 IP 的请求后，再还原到真实域名并按规则转发。

优点：

- 兼容性好
- 配置简单
- 适合大多数家庭路由器

适合：

- 初次使用
- mihomo 稳定版
- 普通透明代理场景

## TUN 模式

TUN 模式通过虚拟网卡接管流量。

要求：

- 系统存在 `/dev/net/tun`
- 容器或虚拟机环境需要宿主机允许 TUN
- 内核和防火墙规则必须配合正常

优点：

- 接管能力更完整
- 对部分应用更友好

风险：

- 环境依赖更高
- 配置错误时可能导致国内外都不通

## Mixed 模式

Mixed 是 TUN 的一种网络栈模式。它仍然依赖 TUN，但使用 mixed stack。

适合：

- system / gVisor 表现不稳定时尝试
- 特定虚拟化环境下改善兼容性

## 降级运行

当用户选择 TUN 或 Mixed，但设备不支持 TUN 时，Clashoo 会尝试自动降级。

降级运行表示：

- 用户配置想用 TUN
- 实际环境无法使用 TUN
- Clashoo 改用兼容模式继续运行

这不是严重错误，但说明当前设备不满足 TUN 条件。

## 选择建议

- 不确定选什么：用 `Fake-IP`
- 需要完整接管：测试 `TUN`
- TUN 有问题：测试 `Mixed`
- 容器环境：先确认宿主机已允许 TUN
