# 名词解释

## LuCI

OpenWrt 的 Web 管理界面。Clashoo 的页面运行在 LuCI 中。

## mihomo

Clash Meta 的延续项目，使用 YAML 配置，兼容大量 Clash 订阅。

## sing-box

通用代理平台，使用 JSON 配置。和 mihomo 不是同一种配置格式。

## Smart

mihomo 的特殊版本，支持 Smart 策略组和相关策略能力。

## Fake-IP

DNS 返回虚拟 IP，由代理核心根据虚拟 IP 找回域名并代理访问。

## Redir-Host

DNS 返回真实 IP，核心按真实目标进行透明代理。兼容方式和 Fake-IP 不同。

## Redirect

常用于 TCP 透明代理，把 TCP 流量重定向到核心监听端口。

## TProxy

常用于 UDP 透明代理，可以保留原始目标地址。

## TUN

虚拟网卡方式。核心通过 TUN 接管系统流量。

## 网络栈

TUN 模式下数据包的处理方式，可选 `gVisor`、`System`、`Mixed`。gVisor 兼容性好但开销略高，System 直接用内核栈，Mixed 是两者的折中。运行模式选「Mixed」即 TUN + Mixed 网络栈。

## Mixed Port

同时支持 HTTP 和 SOCKS 的代理端口，常见端口是 `7890`。

## 显式代理

客户端明确设置代理地址，例如 `127.0.0.1:7890`。

## 透明代理

客户端不设置代理，由路由器自动劫持和转发流量。

## IPv6 DNS 劫持

拦截局域网设备发出的 IPv6 DNS 查询，并交给 Clashoo 内核处理。它不等于开启 IPv6 解析，详细区别见 [[DNS设置]]。

## GeoIP

IP 地理位置数据库，常用于判断 IP 是否属于中国大陆。

## GeoSite

域名规则集，常用于判断域名类别，例如 CN、Google、YouTube。

## 大陆白名单

用于识别中国大陆 IP 或域名，通常用于直连规则。

## Dashboard

代理核心的管理面板，例如 Zashboard、YACD、MetaCubeXD。

## ECS

EDNS Client Subnet。在 DNS 请求里带上一个网段，告诉上游 DNS“客户端大概在哪”，以便 CDN 返回更近的 IP。Clashoo 默认留空（不写入）；要填一般填你自己的本地 / 公网网段，详见 [[DNS设置]]。

## 仅内核

进阶开关。开启后 Clashoo 只拉起内核、原样运行你导入的配置，不接管防火墙 / DNS / 路由，由配置自己完成透明代理。详见 [[仅内核]]。

## 健康检查

Clashoo 用 DNS、显式代理和透明代理测试判断服务是否真的可用。
